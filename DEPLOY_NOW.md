## 🚀 Следующие шаги:

### Шаг 1: Откройте Railway

Перейдите на: **https://railway.app**

### Шаг 2: Войдите через GitHub

Нажмите **"Login with GitHub"** и авторизуйтесь

### Шаг 3: Создайте новый проект

1. Нажмите **"New Project"**
2. Выберите **"Deploy from GitHub repo"**
3. Найдите и выберите репозиторий: **radmir2006/techstore**
4. Railway автоматически начнет деплой

### Шаг 4: Добавьте PostgreSQL

1. В вашем проекте нажмите кнопку **"+ New"**
2. Выберите **"Database"**
3. Выберите **"Add PostgreSQL"**
4. Дождитесь создания базы данных (1-2 минуты)

### Шаг 5: Настройте переменные окружения

1. Кликните на ваш сервис (не на PostgreSQL)
2. Перейдите на вкладку **"Variables"**
3. Нажмите **"+ New Variable"** и добавьте следующие переменные:

#### Сгенерируйте секретный ключ:

Откройте терминал на вашем компьютере и выполните:

```bash
openssl rand -base64 32
```

Скопируйте результат (это будет ваш `NEXTAUTH_SECRET`)

#### Добавьте эти переменные:

```
NEXTAUTH_SECRET=<вставьте_сгенерированный_ключ>
ADMIN_PASSWORD=<придумайте_надежный_пароль>
ADMIN_EMAIL=admin@techstore.ru
NEXT_PUBLIC_SITE_NAME=TechStore
```

**Важно:** Замените значения в `<>` на реальные!

### Шаг 6: Получите URL вашего сайта

1. После первого деплоя Railway создаст домен
2. Перейдите в **"Settings"** → **"Domains"**
3. Скопируйте URL (будет вида: `https://techstore-production-xxxx.up.railway.app`)

### Шаг 7: Добавьте URL в переменные

Вернитесь в **"Variables"** и добавьте:

```
NEXTAUTH_URL=<ваш_URL_из_шага_6>
NEXT_PUBLIC_SITE_URL=<ваш_URL_из_шага_6>
```

Например:
```
NEXTAUTH_URL=https://techstore-production-xxxx.up.railway.app
NEXT_PUBLIC_SITE_URL=https://techstore-production-xxxx.up.railway.app
```

### Шаг 8: Примените миграции базы данных

#### Вариант A: Через Railway CLI (рекомендуется)

```bash
# 1. Установите Railway CLI
npm install -g @railway/cli

# 2. Авторизуйтесь
railway login

# 3. Подключитесь к проекту
railway link

# 4. Примените миграции
railway run npx prisma db push
```

#### Вариант B: Через веб-интерфейс

1. Откройте ваш сервис в Railway
2. Перейдите в **"Settings"**
3. Найдите **"Build Command"**
4. Измените на:
   ```
   npm install && npx prisma generate && npx prisma db push && npm run build
   ```
5. Нажмите **"Deploy"** → **"Redeploy"**

### Шаг 9: Проверьте работу сайта

1. Откройте ваш URL в браузере
2. Перейдите в админ-панель: `/admin/login`
3. Войдите с учетными данными:
   - **Email:** `admin@techstore.ru`
   - **Пароль:** тот, что указали в `ADMIN_PASSWORD`

## 📊 Полезная информация

### Автоматические деплои

Теперь при каждом `git push` в ветку `main` Railway автоматически обновит ваш сайт.

### Просмотр логов

В Railway Dashboard → выберите ваш сервис → вкладка **"Deployments"** → **"View Logs"**

### Мониторинг

Вкладка **"Metrics"** покажет использование ресурсов (CPU, RAM, Network)

### Управление базой данных

Кликните на PostgreSQL сервис → вкладка **"Data"** для просмотра таблиц

## 🆘 Нужна помощь?

### Ошибка при деплое?
- Проверьте логи в разделе "Deployments"
- Убедитесь, что все переменные окружения установлены

### Сайт не открывается?
- Проверьте, что деплой завершился успешно (зеленая галочка)
- Убедитесь, что PostgreSQL запущен

### Ошибка 500?
- Проверьте, что миграции применены
- Проверьте логи приложения
